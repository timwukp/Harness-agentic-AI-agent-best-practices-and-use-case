## Root Cause Analysis

### Bug 1 — Error message color is GREEN instead of RED
**Location:** CSS rule for `.error-message` (line 18)
```css
.error-message { color: green; ... }
```
The `.error-message` class was mistakenly given `color: green` — the same value used by `.success-message`. Since this element has a red-tinted background (`#ffe0e0`), the intent is clearly to display an error in **red**. The fix is a single word change: `green` → `red`.

---

### Bug 2 — Error text says "Internal server error" instead of "Invalid username or password"
**Location:** JavaScript `else` branch handling failed login (line 67)
```js
errorMsg.textContent = 'Internal server error. Please try again later.';
```
This is a client-side credential check (comparing against hardcoded `admin`/`password123`). A failed match is a credentials error, not a server error. The message must reflect the actual failure: invalid credentials.
